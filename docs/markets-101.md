## 2. A 股：制度与工程细节

### 2.1 交易机制
- **集合竞价**：09:15-09:25 报单累计后一次成交；14:57-15:00 尾盘也是集合竞价。
- **连续竞价**：09:30-11:30 / 13:00-15:00（不含 14:57-15:00）。
- **T+1**：今日买入不能当日卖出；但若昨日买入今日已成持仓，可当日卖出。
- **涨跌停板**：主板 ±10%，ST ±5%，创业板 / 科创板 / 北交所 ±20%。涨停价挂单按时间优先排队，到收盘仍未成交即失效。
- **停牌**：常见情形——临时公告、长期停牌（重组）、ST/*ST 特殊处理、盘中 10% 偏离二次停牌（科创板 15%）。

### 2.2 费用结构
回测必须至少加入以下三项：
- **佣金**：2.5 bps（万 2.5）是 2026 零售主流，最低 5 元 / 笔。
- **印花税**：卖出 1 bps（2023 年减半后为 5 bp → 2023/8 起 1 bp；以最新公告为准 ⚠️）。买入不征。
- **过户费**：0.2 bps 双边（沪深合并后的总额）。
- **市场冲击成本**：中等流动性股票 10-30 bps 起步；小盘股显著更高。

### 2.3 数据与复权
- **A 股复权是新手第一坑**：分红/送转/配股/合股/拆股都要调整。常见两种：
  - **前复权**：把历史价格按最新因子调整；画图连续；因子前置但近期为真价。
  - **后复权**：把新价格按最早因子调整；长期持有真实累计收益。
- 回测绝不能混用。统一用**前复权 + 原始价格**双存储，分红事件单独记录。
- 停牌日在价格序列里会缺行，要么 forward-fill 保持市值，要么显式标记为 NaN。

### 2.4 程序化对接差异
- 散户 API：[Tushare Pro](https://tushare.pro/)（需积分/付费）、[AKShare](https://akshare.akfamily.xyz/)（抓取稳定性一般）、QMT/Ptrade（券商私有）。
- 实盘走券商 API（如 QMT、EasyTrader、量投 QuanTrade），延迟普遍 50-200 ms；高频难做。
- **不能**随便拉 tick 级数据；Level-2 数据在上交所/深交所都是收费的。

## 3. 美股：制度与工程细节

### 3.1 交易时段与流动性
- **常规时段**：09:30-16:00 ET，流动性最高。
- **盘前（pre-market）**：04:00-09:30 ET，流动性稀薄 → 点差大、滑点高。
- **盘后（after-hours）**：16:00-20:00 ET，同样流动性不好。
- 程序化策略**默认仅在常规时段成交**；若策略涉及业绩公告后反应，务必专门处理盘后数据。

### 3.2 费用结构
- **零售佣金**：Robinhood / Schwab / Fidelity 已经 0 佣金；IBKR Fixed 仍按 $0.005/股，最高 1% notional 封顶。
- **SEC Fee**：卖单征收，2026 年 ~0.278 bps（定期调整 ⚠️）。
- **FINRA TAF**：卖单 $0.000166/股。
- 零佣金不代表零成本：Payment for Order Flow (PFOF) 实际让你在价差上吃亏，典型 2-4 bps。

### 3.3 数据陷阱
- **Dividends / Splits**：yfinance 的 `Adj Close` 已经调整，但 `Dividends`/`Stock Splits` 要单独处理才能重建原始价。
- **退市与并购**：标普 500 历史成分每年都变；生存者偏差必须用含退市的全量历史。
- **Adjusted volume**：拆股后历史 volume 要反向缩放才能比较，yfinance 会自动处理，但 Alpha Vantage 的 `adjusted` 接口字段含义不一样，**要读 docstring 不要凭感觉**。

### 3.4 程序化对接
- 免费档：yfinance（非官方）、Alpha Vantage（每天 500 次）、Polygon Starter。
- 付费档：Polygon Stocks Developer $29/m（分钟 + tick 回放）、IEX Cloud（已宣布 2024 关停，转用其他源）。
- 实盘：IBKR TWS API / Alpaca / TD Ameritrade（IBKR 成为并购后主力）；延迟 200-1000 ms（跨太平洋）。

## 4. 期货（中国商品期货）

### 4.1 交易机制
- **合约乘数**：每手 = 乘数 × 价格，保证金 = notional × 保证金率（各品种不同，通常 5-15%）。
- **主力合约切换（换月）**：持仓主力品种每 1-3 个月换月，持仓量与成交量主力合约不是同一天切换；回测必须构造**连续合约**（Panama / Ratio / Calendar 法），否则价格曲线会出现假跳空。
- **夜盘**：上期所黑色系/能源化工、郑商所多品种均有 21:00-23:00 或 23:30 夜盘。隔夜保证金按闭市价计算。
- **交割**：临近交割月散户不得持仓；机构可走实物交割（工业品）或现金交割（股指）。

### 4.2 费用结构
- **手续费**：按固定金额（如螺纹 3 元/手）或 bps（股指 0.23 bps-2.3 bps 不等）。
- **平今仓**：**日内平仓**部分品种（如螺纹、铁矿）手续费**放大 2-6 倍**（打击日内短炒）。回测的 CTA 策略若高日内频率要单列该条目。
- **保证金**：交易所保证金 + 期货公司加成；加成部分可议价。

### 4.3 数据坑
- **主力换月**：如果直接拼接主力合约，价格会跳空（新主力的价格与旧主力的差价即 **Roll Yield**）。正确做法是：
  1. 使用期货交易所的「指数」或「连续合约」。
  2. 或自制 Panama 连续：换月时把历史价格全部减去/加上价差 `old - new`。
  3. 保留 "raw" 与 "adjusted" 两套以便审计。
- **交易所品种不同时段**：上期所、郑商所、大商所、中金所夜盘开平时间不完全一致。

## 5. 期权：感性入门

### 5.1 核心名词
- **Call/Put**：看涨/看跌；**行权价（Strike）**；**到期（Expiry）**；**内在价值 = max(0, S-K)（看涨）**；**时间价值 = 权利金 - 内在价值**。
- **隐含波动率 IV**：把期权市场价反算回 BS 得到的波动率，是市场对未来波动的预期。
- **Greeks**：
  - **Delta** ≈ 标的每动 1%，权利金动多少比例。
  - **Gamma**：Delta 的变化率，衡量对冲频率压力。
  - **Vega**：波动率每动 1%，权利金动多少。
  - **Theta**：每天时间价值衰减多少。
  - **Rho**：利率敏感度（相对次要）。
- **波动率微笑**：同一到期日、不同行权价 IV 不同——远离当前价的两端 IV 通常更高（微笑曲线）。

### 5.2 交易所与标的
- **中国**：上证 50ETF 期权（510050）、沪深 300ETF 期权（510300/159919）、中证 500ETF、创业板 50 ETF、股指期权（IO/MO）。
- **美国**：最大流动性在 SPX、SPY、QQQ、AAPL 等单股；CBOE 是做市主场。
- **加密**：Deribit（BTC/ETH 期权流动性最好），OKX 也在扩。

### 5.3 费用与摩擦
- 佣金：券商通常每张 1.3-10 元（CN）或 $0.65/张（US）。
- **点差 + 市场冲击**：Out-of-the-money 合约点差可能 5-15%，比佣金贵得多；回测务必按 mid-price 加半点差保守估算。
- **行权相关**：美式期权可提前行权；欧式不能。ETF 期权多为欧式（CN）或美式（US）。

## 6. 加密货币

### 6.1 市场结构
- **CEX（中心化交易所）**：Binance / OKX / Bybit / Coinbase。订单簿撮合，接口统一度高（可用 `ccxt`）。
- **DEX**：Uniswap / Curve / Jupiter。需要写 Web3 RPC，不在入门阶段重点。
- **产品线**：
  - **现货（Spot）**：BTC/USDT。
  - **U 本位永续合约（USDT-margined Perp）**：最主流衍生品。
  - **币本位合约（Coin-margined）**：用标的币做保证金，收益曲线更剧烈。
  - **期权**：Deribit 为主。
  - **Spot Margin**：相对少用。

### 6.2 费用与摩擦
- **Maker/Taker**：Binance 零售 taker 10 bps，maker 10 bps；VIP 0/1 bps；BNB 抵扣再 25% 折扣。
- **Withdraw Fee**：链上提币手续费随链条拥堵波动（ETH 可到数十刀/次）。
- **资金费率（Funding Rate）**：永续合约每 8 小时结算一次，fee = rate × notional。牛市常为正（多付给空），长期拿多头成本可观。回测 **必须** 把资金费加入成本。

### 6.3 数据特性
- `ccxt` 提供统一 OHLCV、订单簿、行情、历史 funding；但 ccxt 不同版本接口偶尔漂移，需要在项目里锁定版本。
- 限频：Binance Weight 限制、OKX 更细粒度、Bybit 有 IP 与账户双限；并发请求一定要按交易所的 weight 做节流。
- **稳定币不稳定**：2022 年 LUNA/UST 崩盘、2023 年 USDC 脱锚均造成 5-10% 价差。回测要把 USDT 当成 1.0，但模型要对突发风险有单独测试。

## 7. 统一的成本计量模型

任何回测在计算净收益时至少包含这三项：

```
net_return = gross_return
           - commission_bps / 10_000 × turnover
           - slippage_bps    / 10_000 × turnover
           - impact_bps      / 10_000 × sqrt(participation_rate)
           - funding_bps     / 10_000 × hold_periods       (仅永续)
           - stamp_duty      / 10_000 × sell_notional      (仅 A 股)
```

项目里对应实现见 `src/quant_lucky/costs/`：
- 各市场 `CostModel`（A 股 / 美股 / 加密现货 / 永续 / 期货）
- `SlippageModel`（固定 bps / 半点差 / 平方根冲击）
- `AnnualCostCalculator`（换手率 × 频率 → 年化拖累）

## 8. 坑点汇总（持续更新，见 `docs/pitfalls.md`）

| 市场 | 坑 | 解决 |
|------|----|------|
| A 股 | 复权因子漏算 | 前复权 + 原始价格双存储 |
| A 股 | 涨停时假装成交 | 回测撮合前显式过滤涨停股 |
| A 股 | 停牌空值 | forward-fill 或 NaN 并在持仓模块处理 |
| 美股 | 盘前盘后低流动性当常规时段 | 默认仅撮合 09:30-16:00 |
| 美股 | 生存者偏差 | 使用含退市股的全量历史池 |
| 期货 | 主力拼接价差 | Panama 连续合约 + 保留原始 |
| 期货 | 平今手续费放大 | 每笔分 "open/close_today/close" 记账 |
| 期权 | 仅看 Delta | 波动率、Theta、Vega 同样要算 |
| 加密 | 资金费被忽略 | 永续回测强制计入 funding_rate |
| 加密 | 限频被 ban | ccxt Weight 节流 + 指数退避 |
| 全部 | 零成本回测 | 强制最低滑点，任何 "无成本" 结果加 warning |
